## هيكل المشروع

```
wameed/
├── windows-receiver/
│   ├── src/
│   │   ├── receiver.py        ← المستقبل (واجهة + WebSocket + UDP)
│   │   ├── requirements.txt   ← مكتبات Python (نسخ مثبتة)
│   │   └── wameed.ico
│   ├── installer/
│   │   ├── wameed.iss         ← سكربت Inno Setup
│   │   ├── LICENSE.txt
│   │   └── README-install.md
│   └── scripts/
│       ├── build.bat          ← بناء exe + مثبّت (خطوة واحدة)
│       └── dev_run.bat        ← تشغيل تطويري
│
└── app/src/main/
    ├── AndroidManifest.xml
    └── java/com/wameed/
        ├── SplashActivity.kt  ← شاشة البداية بتأثير وميض
        ├── MainActivity.kt    ← الشاشة الرئيسية (اتصال + سجل + إعدادات)
        ├── ShareActivity.kt   ← يستقبل المشاركة ويُرسل فوراً
        ├── WameedSender.kt    ← يرسل للكمبيوتر عبر WebSocket
        ├── WameedServer.kt    ← يستقبل من الكمبيوتر (Ktor WebSocket server)
        ├── WameedConnectionService.kt ← خدمة الخلفية (keep-alive + receiver)
        ├── WameedNsdHelper.kt ← تسجيل mDNS/NSD لاكتشاف الهاتف
        ├── WameedEvents.kt    ← أحداث SharedFlow بين المكونات
        ├── WameedPrefs.kt     ← يحفظ الإعدادات + سجل الملفات
        ├── FileSaver.kt       ← حفظ الملفات المستلمة في Downloads
        ├── ReceiveActivity.kt ← واجهة الاستقبال + طلب الاقتران
        └── DeviceDiscovery.kt ← اكتشاف الكمبيوتر تلقائياً (UDP)
```

## تشغيل الكمبيوتر (Windows)

### للمستخدم النهائي (مستحسن)
1. حمّل أحدث `WameedSetup-x.y.z.exe` من صفحة الإصدارات.
2. ثبّت بنقرة واحدة — يتولى المثبّت: اختصارات، Firewall، Autostart.
3. شغّل **Wameed** من قائمة ابدأ.
4. لا حاجة لتثبيت Python.

### للمطوّر
```
cd windows-receiver
scripts\dev_run.bat      # يشغّل receiver.py مع لوج حي
```
يتطلب Python 3.8+.

### لبناء المثبّت (للإصدارات)
```
cd windows-receiver
scripts\build.bat
# يُنتج: dist\Wameed.exe  +  installer\Output\WameedSetup-1.0.0.exe
```
يتطلب: Python + Inno Setup 6 (https://jrsoftware.org/isdl.php).

### حالات الاتصال في نافذة ويندوز
- 🔵 **الخادم يعمل • بانتظار الهاتف** — جاهز للاستقبال.
- 🟢 **الهاتف متصل الآن** — جلسة نشطة من الهاتف.
- 🔴 **خطأ** — منفذ مستخدم أو مشكلة.

### الإعدادات المتاحة (Windows)
| الإعداد | الخيارات |
|---------|---------|

| مجلد الحفظ | افتراضي: `Downloads/Wameed` أو أي مجلد |
| المنفذ | افتراضي: `7788` |

## تشغيل الهاتف (Android)

### المتطلبات
- Android Studio: https://developer.android.com/studio
- هاتف أندرويد (Android 8.0+) — يدعم Samsung Galaxy، Huawei، وجميع أجهزة Android

### الخطوات
1. افتح Android Studio → افتح مجلد المشروع
2. انتظر Gradle sync
3. اضغط Run (▶) على هاتفك
4. ستظهر شاشة وميض مع تأثير Flash
5. التطبيق يبحث تلقائياً عن الكمبيوتر أو أدخل IP يدوياً

### صفحات التطبيق
- **الاتصال** — اكتشاف الأجهزة + إرسال ملفات
- **السجل** — قائمة الملفات المرسلة (نجاح/فشل)
- **الإعدادات** — وضع العرض، معلومات الاتصال

## طريقة الاستخدام

### إرسال من الهاتف → الكمبيوتر
1. **الكمبيوتر:** شغّل `Wameed.exe` (أو من قائمة ابدأ) ← يعمل في الخلفية بدون نافذة Terminal
2. **الهاتف:** افتح أي ملف أو صورة أو رابط
3. اضغط **مشاركة** → اختر **⚡ وميض**
4. **أول مرة فقط:** يظهر على الكمبيوتر طلب «السماح لـ <اسم الهاتف> بالإرسال؟» — اضغط «✓ السماح دائماً».
5. يفتح الملف على الكمبيوتر فوراً ✓ (لا يُطلب الاقتران مرة أخرى من نفس الهاتف)

### إرسال من الكمبيوتر → الهاتف
1. **الهاتف:** تأكد أن وميض مفتوح ومتصل بالكمبيوتر (أو اضغط زر "📡 استقبال").
2. **الكمبيوتر:** اضغط **📤 إرسال للهاتف** → اختر ملف أو اكتب نص → اختر الجهاز → اضغط ⚡ إرسال.
3. **أول مرة فقط:** يظهر على الهاتف طلب اقتران — اضغط «السماح».
4. الملف يُحفظ في مجلد `Downloads/Wameed` على الهاتف ✓

> **ملاحظة:** وضع الاستقبال يبقى مفعلاً لمدة 5 دقائق حتى لو خرجت من التطبيق مؤقتاً.

## الأمان (v1.1.0)

- **اقتران مرة واحدة لكل هاتف**: الكمبيوتر يحفظ الأجهزة الموثوقة في `%USERPROFILE%\.wameed\trusted_devices.json` بمعرّف فريد لكل هاتف.
- **إدارة الثقة**: الإعدادات → «الأجهزة الموثوقة» → زر إلغاء الثقة لكل جهاز أو الكل.
- **الهواتف المرفوضة/غير المقترنة** لا يمكنها إرسال أي شيء (الـ `ping` مسموح فقط للتحقق من الاستماع).

## استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| لا يمكن الاتصال | تأكد أن الهاتف والكمبيوتر على نفس Wi-Fi |
| Windows Firewall يحجب | السماح لـ Python في Firewall |
| لم يُعثر على أجهزة | استخدم "إدخال يدوي" وأدخل IP:Port |
| أيقونة التطبيق لا تظهر | أعد تثبيت التطبيق (uninstall ثم install) |

# وميض — قدرات التطبيق (Skills)

> ملخص شامل لجميع ميزات وميض وطريقة عملها تقنياً.

---

## 1. الإرسال من الهاتف → الكمبيوتر

### الملفات المعنية
| الجانب | الملفات |
|--------|---------|
| هاتف (مُرسِل) | `WameedSender.kt`, `ShareActivity.kt`, `MainActivity.kt` |
| كمبيوتر (مُستقبِل) | `receiver.py` — WebSocket server على المنفذ **7788** |

### كيف يعمل
1. **الاكتشاف**: الكمبيوتر يبث UDP broadcast كل 2 ثانية على المنفذ **7789**.
2. **الهاتف** يستمع على نفس المنفذ ويكتشف الكمبيوتر تلقائياً (`DeviceDiscovery.kt`).
3. **المشاركة**: المستخدم يشارك ملف/صورة/رابط من أي تطبيق → `ShareActivity` يلتقطه.
4. **الاتصال**: `WameedSender` يفتح WebSocket قصير العمر للكمبيوتر (منفذ 7788).
5. **الاقتران (مرة واحدة)**: أول اتصال → الكمبيوتر يعرض نافذة "السماح؟" → بعد الموافقة يُحفظ في `trusted_devices.json`.
6. **النقل**: يُرسل `hello` → `file_meta` → binary chunks (1MB) → الكمبيوتر يحفظ ويفتح.

### البروتوكول
```
Phone → PC:  {"type": "hello", "device": "Galaxy S24", "device_id": "uuid-..."}
PC → Phone:  {"status": "paired"}  أو  {"status": "pairing_required"}
Phone → PC:  {"type": "file_meta", "filename": "photo.jpg", "size": 1234567, "chunks": 2}
Phone → PC:  [binary chunk 1]
Phone → PC:  [binary chunk 2]
PC → Phone:  {"status": "saved", "path": "C:\\...\\photo.jpg"}
```

## 2. الإرسال من الكمبيوتر → الهاتف

### الملفات المعنية
| الجانب | الملفات |
|--------|---------|
| كمبيوتر (مُرسِل) | `receiver.py` — `WameedSender` class + واجهة "إرسال للهاتف" |
| هاتف (مُستقبِل) | `WameedServer.kt` — WebSocket server على المنفذ **7789** |
| خدمة الخلفية | `WameedConnectionService.kt` — تبقي السيرفر حياً |
| عرض الاستقبال | `ReceiveActivity.kt` — واجهة الاستقبال + طلب الموافقة |

### كيف يعمل
1. **تفعيل الاستقبال**: عند اتصال الهاتف بالكمبيوتر، تُفعَّل خدمة `WameedConnectionService` التي تشغل `WameedServer` (Ktor CIO) على المنفذ **7789**.
2. **اكتشاف الهاتف**: الهاتف يسجل خدمة mDNS/NSD (`_wameed._tcp`) عبر `WameedNsdHelper`. الكمبيوتر يكتشفه عبر `zeroconf` أو من قائمة الأجهزة الموثوقة.
3. **الإرسال**: المستخدم يضغط "📤 إرسال للهاتف" على الكمبيوتر → يختار ملف أو نص → يحدد الجهاز → يضغط إرسال.
4. **الاتصال**: الكمبيوتر يفتح WebSocket للهاتف (منفذ 7789).
5. **الاقتران**: الهاتف يعرض شاشة موافقة عبر `ReceiveActivity`.
6. **النقل**: نفس البروتوكول (hello → file_meta → chunks → saved).
7. **الحفظ**: `FileSaver.kt` يحفظ الملف في `Downloads/Wameed` عبر MediaStore.

### البروتوكول
```
PC → Phone:  {"type": "hello", "device": "DESKTOP-ABC", "device_id": "pc-DESKTOP-ABC"}
Phone → PC:  {"status": "paired"}
PC → Phone:  {"type": "file_meta", "filename": "report.pdf", "size": 5000000, "chunks": 5}
PC → Phone:  [binary chunk 1..5]
Phone → PC:  {"status": "saved"}
```

### استمرارية الاستقبال (تلقائي بالكامل)
- **لا يوجد زر "استقبال" يدوي** — الاستقبال يعمل تلقائياً عند أي اقتران ناجح.
- **عند الاتصال**: `WameedConnectionService` تشغل `WameedServer` تلقائياً.
- **عند الخروج المؤقت**: السيرفر يبقى مفعلاً لمدة **5 دقائق** حتى لو خرج المستخدم من التطبيق.
- **عند العودة**: `onResume()` يُعيد ضبط المؤقت ويُعيد تشغيل السيرفر إن كان قد توقف.
- **إيقاف**: فقط زر "إيقاف" في الإشعار أو إغلاق التطبيق كلياً.

### الملفات المستلمة (تبويب "المستلمات")
- تبويب مستقل في التطبيق يعرض كل الملفات المحفوظة في `Downloads/Wameed`.
- يقرأ الملفات عبر MediaStore (Android 10+) أو من الملفات مباشرة (أقدم).
- كل ملف يعرض: الاسم، الحجم، تاريخ التعديل، أيقونة حسب النوع.
- أزرار: **فتح** (ACTION_VIEW) + **مشاركة** (ACTION_SEND).
- زر "فتح مجلد وميض" لفتح المجلد في مدير الملفات.

## 3. إرسال النصوص

### من الهاتف → الكمبيوتر
- `WameedSender.kt` يرسل `{"type": "text", "text": "..."}`.
- الكمبيوتر ينسخه للحافظة + يعرض Toast.

### من الكمبيوتر → الهاتف
- `receiver.py` — `WameedSender.send_text()` يرسل `{"type": "text", "text": "..."}`.
- الهاتف يعرضه في `ReceiveActivity` مع زر "نسخ".

---

## 4. إرسال الروابط

### من الهاتف → الكمبيوتر
- `{"type": "url", "url": "https://..."}` → الكمبيوتر يفتحه في المتصفح.

### من الكمبيوتر → الهاتف
- `{"type": "url", "url": "https://..."}` → الهاتف يعرضه مع زر "فتح".

---

## 5. الاكتشاف التلقائي

| الاتجاه | الآلية |
|---------|--------|
| الهاتف يكتشف الكمبيوتر | UDP broadcast من الكمبيوتر كل 2 ثانية على منفذ **7789** |
| الكمبيوتر يكتشف الهاتف | mDNS/NSD (`_wameed._tcp`) عبر `zeroconf` (Python) و `WameedNsdHelper` (Android) |

---

## 6. الأمان والاقتران

- **أول اتصال**: طلب اقتران يظهر على الطرف المستقبل (نافذة على الكمبيوتر أو شاشة على الهاتف).
- **بعد الموافقة**: يُحفظ الجهاز كـ "موثوق" ولا يُسأل مجدداً.
- **ملفات الثقة**:
  - كمبيوتر: `%USERPROFILE%\.wameed\trusted_devices.json`
  - هاتف: SharedPreferences عبر `WameedPrefs`
- **الأجهزة غير الموثوقة**: لا يمكنها إرسال أي محتوى (فقط `ping` مسموح).

---

## 7. خدمة الخلفية (Keep-Alive)

`WameedConnectionService` — خدمة Foreground تعمل في الخلفية:

| الوظيفة | التفاصيل |
|---------|----------|
| WS keep-alive | اتصال WebSocket مفتوح مع الكمبيوتر + ping كل 25 ثانية |
| سيرفر الاستقبال | `WameedServer` على منفذ 7789 — يبقى حياً لمدة 5 دقائق بعد إغلاق التطبيق |
| NSD | تسجيل الخدمة ليكتشفها الكمبيوتر |
| إيقاف تلقائي | بعد 5 دقائق من عدم النشاط أو 3 محاولات ping فاشلة |

## 8. سجل النقل (التاريخ)

- كل عملية إرسال أو استقبال تُسجل في `WameedPrefs` مع حقل `direction`:
  - `"sent"` — ملف/نص/رابط أُرسل من الهاتف
  - `"received"` — ملف/نص/رابط استُلم من الكمبيوتر
- **تبويب السجل** يعرض كل العمليات مع رمز اتجاه (`↑` مرسل / `↓` مستلم).
- **تبويب المستلمات** يعرض الملفات الفعلية المحفوظة مع أزرار فتح ومشاركة.

### تبويبات التطبيق
| # | التبويب | الوصف |
|---|---------|-------|
| 0 | الاتصال | حالة الاتصال + أزرار إرسال/تحديث + الأجهزة المكتشفة |
| 1 | السجل | كل عمليات النقل (إرسال + استقبال) مع رمز الاتجاه |
| 2 | المستلمات | الملفات المحفوظة في Downloads/Wameed مع فتح/مشاركة |
| 3 | الإعدادات | وضع العرض + keep-alive + معلومات الاتصال |

## 9. المنافذ المستخدمة

| المنفذ | البروتوكول | الاتجاه | الوظيفة |
|--------|-----------|---------|---------|
| **7788** | TCP/WebSocket | الهاتف → الكمبيوتر | سيرفر الاستقبال على الكمبيوتر |
| **7789** | UDP | الكمبيوتر → الشبكة | اكتشاف تلقائي (broadcast) |
| **7789** | TCP/WebSocket | الكمبيوتر → الهاتف | سيرفر الاستقبال على الهاتف |

## 10. هيكل المشروع

```
wameed/
├── windows-receiver/src/
│   ├── receiver.py          ← كل شيء: واجهة + WS server + WS sender + UDP
│   └── requirements.txt
│
└── app/src/main/java/com/wameed/
    ├── MainActivity.kt       ← الشاشة الرئيسية (Compose)
    ├── ShareActivity.kt      ← استقبال المشاركة من تطبيقات أخرى
    ├── SplashActivity.kt     ← شاشة البداية
    ├── WameedSender.kt       ← إرسال للكمبيوتر (WS client)
    ├── WameedServer.kt       ← استقبال من الكمبيوتر (WS server / Ktor)
    ├── WameedConnectionService.kt  ← خدمة الخلفية (keep-alive + receiver)
    ├── WameedNsdHelper.kt    ← تسجيل mDNS/NSD
    ├── WameedEvents.kt       ← أحداث SharedFlow
    ├── WameedPrefs.kt        ← إعدادات + سجل
    ├── DeviceDiscovery.kt    ← اكتشاف الكمبيوتر (UDP listener)
    ├── FileSaver.kt          ← حفظ الملفات في Downloads
    └── ReceiveActivity.kt    ← واجهة الاستقبال + اقتران
```
